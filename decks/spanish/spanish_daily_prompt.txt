  I might also explicitly provide text in brackets with an instruction for you to par

    Example "the sentence is {{big::not small}} | grande o largo tambien".
  In this case - simply add it to the back in addition to what you've generated normally. In other words: no logic - trivial concatenation with a <br>

CARD GENERATION STRATEGY
────────────────────────

1️⃣ Adaptive Card Types

Generate only what improves retention:

Production (EN → ES)
Use when:
- Active recall improves fluency
- Expression is high-frequency
- Structure is reusable

Pattern
Use when:
- A structure needs abstraction
- Formula thinking improves transfer

Cloze (ONLY when efficient)
Use cloze if hiding something improves encoding OR if EXPLICIT CLOZE is present
- irregular conjugations
- pronoun placement
- prepositions (por/para, a/en)
- tense contrasts
- dialect-sensitive forms
- contractions (al, del)
- superlatives, stem changes
- EXPLICIT CLOZE
  ALWAYS include my cue if provided.
    You can build upon it & expand it as for other cards, but don't make it shorter. Again, just append your variant to the cue.
  If I've provided some additoinal information within bracketets within EXPLICIT CLOZE - treat it as a subprompt when preparing the cue for the given card.
- Cloze format: https://docs.ankiweb.net/editing.html#cloze-deletion
  In essence:
    if one cloze field -> wrap it in {{c1::cloze}}
    if more -> {{c1::cloze}} {{c2::cloze}} ... {{cN::cloze}}
    if cue -> {{c1::cloze::cue}}
- If I don't provide a cue explicitly and it is benefitial - generate one.

Do NOT force cloze.

⚠️ Do NOT generate recognition cards (Spanish → English).
  They are intentionally excluded because passive recognition provides little learning value compared to production.
